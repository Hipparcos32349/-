# 项目介绍
系统依托YOLO_V8+MiDaS深度视觉模型以及GCN图神经网络，实现学生状态量化识别，自动统计逃课离席人数，结合大模型生成学情报告，解决课堂出勤难统计、学生违纪难实时监管、学情复盘缺量化数据、课堂管理缺少客观分析依据的教学痛点。

使用说明
# 一、运行前环境准备
1. 确保已安装依赖
本项目需要以下核心库：
Python 3.8+
OpenCV
PyTorch
Ultralytics YOLO
Torch Geometric（GCN 图网络）
NumPy
Requests
2. 准备视频文件
在代码同一目录下，放入你的课堂视频
支持格式：.mp4 / .avi / .mov 等常见视频格式
确认代码中视频名称与实际文件名一致
video_path = "5月25日.mp4"
3. 确认大模型 API 可用
本项目的火山方舟 API 已内置为作者本人的api，联网即可自动生成学情报告

# 二、项目运行步骤
步骤 1：打开代码运行工具
使用 VS Code / PyCharm / 命令行 均可运行本项目。
步骤 2：运行主程序
直接点击运行按钮，或在终端输入：
python main_2.py
步骤 3：观察启动日志
正常启动会显示：
✅ 教室智能分析系统（YOLO+深度+GCN+离席检测）已启动
若出现 GCN 权重加载失败，不影响使用，仅提示未加载预训练模型，系统会自动切换为安全模式。

# 三、界面说明（运行后出现两个窗口）
窗口 1：Camera（课堂实时分析画面）
绿色框：正常上课的学生
红色框 + LEAVE：检测到站起、离席、逃课的学生
左上角文字：
Total:xx：当前画面总学生人数
Leave:xx：当前离席人数
窗口 2：Depth（课堂深度热力图）
显示教室空间深度信息
用于提升 GCN 空间分析精度
无需操作，仅作可视化展示

# 四、功能使用方法
1. 自动检测（全程无需操作）
程序会自动完成：
学生目标检测（YOLO）
单目深度计算（MiDaS）
空间图网络分析（GCN）
离席 / 站立 / 逃课行为判定
人数统计与状态标记
2. 生成专业学情报告（核心功能）
当视频播放到你需要分析的画面时：按下键盘 s 键（save/report 快捷键）
控制台会立即输出：
============================================================
📝 课堂学情分析报告
============================================================
随后自动生成专业教学反思报告，包含：
出勤情况详细分析
课堂纪律分析
可直接使用的教学建议
（由多模态大模型自动生成）
3. 退出程序
按下键盘 q 键，关闭窗口并结束程序。

# 五、视频播放完毕
视频播放结束后会自动停止，并提示：
✅ 视频播放完毕

# 六、使用注意事项
视频必须是教室正面视角，效果最佳
画面光线充足，学生无大面积遮挡
运行时保持联网，才能生成 AI 报告
若画面卡顿，属于正常现象（深度模型计算量大）
检测人数、离席状态均为实时统计，按s可截取当前帧生成报告
